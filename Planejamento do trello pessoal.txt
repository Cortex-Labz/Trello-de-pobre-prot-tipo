Planejamento Completo - Clone do Trello

  1. STACK TECNOLÓGICA RECOMENDADA

  Backend:
  - Node.js + Express (ou NestJS para maior estrutura)
  - PostgreSQL (banco relacional para integridade dos dados)
  - Prisma ORM (ou TypeORM)
  - Socket.io (atualizações em tempo real)
  - JWT (autenticação)
  - Redis (cache e sessões)
  - AWS S3 / Cloudinary (armazenamento de anexos)

  Frontend:
  - React + TypeScript
  - React Beautiful DnD (drag and drop)
  - Tailwind CSS (ou Material-UI)
  - React Query (gerenciamento de estado servidor)
  - Zustand (estado global)
  - Socket.io Client (tempo real)
  - React Router (navegação)

  3. ESTRUTURA DO BANCO DE DADOS

  -- USUÁRIOS E AUTENTICAÇÃO
  users
    - id (UUID)
    - email (unique)
    - password_hash
    - name
    - avatar_url
    - created_at
    - updated_at

  -- WORKSPACES
  workspaces
    - id (UUID)
    - name
    - description
    - owner_id (FK users)
    - created_at
    - updated_at

  workspace_members
    - id (UUID)
    - workspace_id (FK workspaces)
    - user_id (FK users)
    - role (ADMIN, MEMBER)
    - joined_at

  -- BOARDS
  boards
    - id (UUID)
    - workspace_id (FK workspaces)
    - name
    - description
    - background_color
    - background_image_url
    - visibility (PRIVATE, WORKSPACE, PUBLIC)
    - is_archived
    - created_at
    - updated_at

  board_members
    - id (UUID)
    - board_id (FK boards)
    - user_id (FK users)
    - role (ADMIN, MEMBER, OBSERVER)
    - added_at

  -- LISTS
  lists
    - id (UUID)
    - board_id (FK boards)
    - title
    - position (integer para ordenação)
    - is_archived
    - created_at
    - updated_at

  -- CARDS
  cards
    - id (UUID)
    - list_id (FK lists)
    - title
    - description (text)
    - position (integer)
    - due_date
    - is_completed
    - cover_url
    - is_archived
    - created_by (FK users)
    - created_at
    - updated_at

  card_members
    - id (UUID)
    - card_id (FK cards)
    - user_id (FK users)
    - assigned_at

  -- LABELS
  labels
    - id (UUID)
    - board_id (FK boards)
    - name
    - color

  card_labels
    - card_id (FK cards)
    - label_id (FK labels)

  -- CHECKLISTS
  checklists
    - id (UUID)
    - card_id (FK cards)
    - title
    - position
    - created_at

  checklist_items
    - id (UUID)
    - checklist_id (FK checklists)
    - title
    - is_completed
    - position
    - created_at
    - updated_at

  -- COMENTÁRIOS
  comments
    - id (UUID)
    - card_id (FK cards)
    - user_id (FK users)
    - content (text)
    - created_at
    - updated_at

  -- ATTACHMENTS
  attachments
    - id (UUID)
    - card_id (FK cards)
    - name
    - url
    - file_size
    - mime_type
    - uploaded_by (FK users)
    - uploaded_at

  -- ATIVIDADES (LOG)
  activities
    - id (UUID)
    - board_id (FK boards)
    - card_id (FK cards, nullable)
    - user_id (FK users)
    - action_type (CREATED, UPDATED, MOVED, ARCHIVED, etc)
    - details (JSONB)
    - created_at

  -- NOTIFICAÇÕES
  notifications
    - id (UUID)
    - user_id (FK users)
    - type (MENTION, ASSIGNMENT, DUE_DATE, etc)
    - content
    - related_card_id (FK cards, nullable)
    - is_read
    - created_at

  4. ENDPOINTS DA API (Backend)

  AUTH
  POST   /api/auth/register
  POST   /api/auth/login
  POST   /api/auth/logout
  POST   /api/auth/refresh-token
  POST   /api/auth/forgot-password
  POST   /api/auth/reset-password

  USERS
  GET    /api/users/me
  PUT    /api/users/me
  GET    /api/users/:id
  GET    /api/users/search?q=

  WORKSPACES
  GET    /api/workspaces
  POST   /api/workspaces
  GET    /api/workspaces/:id
  PUT    /api/workspaces/:id
  DELETE /api/workspaces/:id
  GET    /api/workspaces/:id/members
  POST   /api/workspaces/:id/members
  DELETE /api/workspaces/:id/members/:userId

  BOARDS
  GET    /api/boards
  POST   /api/boards
  GET    /api/boards/:id (inclui lists, cards, members)
  PUT    /api/boards/:id
  DELETE /api/boards/:id
  POST   /api/boards/:id/archive
  POST   /api/boards/:id/duplicate
  GET    /api/boards/:id/members
  POST   /api/boards/:id/members
  PUT    /api/boards/:id/members/:userId (atualizar role)
  DELETE /api/boards/:id/members/:userId

  LISTS
  POST   /api/boards/:boardId/lists
  PUT    /api/lists/:id
  DELETE /api/lists/:id
  POST   /api/lists/:id/archive
  PUT    /api/lists/reorder (mudar posição)

  CARDS
  POST   /api/lists/:listId/cards
  GET    /api/cards/:id
  PUT    /api/cards/:id
  DELETE /api/cards/:id
  POST   /api/cards/:id/archive
  PUT    /api/cards/:id/move (mover entre listas)
  PUT    /api/cards/reorder
  POST   /api/cards/:id/duplicate
  GET    /api/cards/:id/members
  POST   /api/cards/:id/members
  DELETE /api/cards/:id/members/:userId

  LABELS
  GET    /api/boards/:boardId/labels
  POST   /api/boards/:boardId/labels
  PUT    /api/labels/:id
  DELETE /api/labels/:id
  POST   /api/cards/:cardId/labels/:labelId
  DELETE /api/cards/:cardId/labels/:labelId

  CHECKLISTS
  POST   /api/cards/:cardId/checklists
  PUT    /api/checklists/:id
  DELETE /api/checklists/:id
  POST   /api/checklists/:id/items
  PUT    /api/checklists/items/:id
  DELETE /api/checklists/items/:id

  COMMENTS
  GET    /api/cards/:cardId/comments
  POST   /api/cards/:cardId/comments
  PUT    /api/comments/:id
  DELETE /api/comments/:id

  ATTACHMENTS
  POST   /api/cards/:cardId/attachments
  DELETE /api/attachments/:id

  ACTIVITIES
  GET    /api/boards/:boardId/activities
  GET    /api/cards/:cardId/activities

  NOTIFICATIONS
  GET    /api/notifications
  PUT    /api/notifications/:id/read
  PUT    /api/notifications/read-all

  SEARCH
  GET    /api/search?q=&type=&boardId=

  WEBSOCKET EVENTS
  - board:updated
  - card:created
  - card:updated
  - card:moved
  - card:deleted
  - comment:created
  - notification:new
